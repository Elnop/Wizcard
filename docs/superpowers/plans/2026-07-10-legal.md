# Légal (RGPD / LCEN) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ajouter les pages légales obligatoires (mentions légales, politique de confidentialité RGPD, CGU) + un footer global qui les référence, avec une identité éditeur centralisée.

**Architecture:** Un fichier de config unique (`legal-config.ts`) porte l'identité éditeur/hébergeur/contact et les valeurs de copie. Un route group `(legal)` héberge trois pages serveur statiques en français, chacune consommant la config et déclarant `generateMetadata` (indexable). Un composant `Footer` global, monté dans le root layout, lie les trois pages + le Discord depuis toute page du site.

**Tech Stack:** Next.js 16 (App Router, React Server Components), TypeScript, CSS Modules. Pas de nouvelle dépendance.

## Global Constraints

- **Français** pour tout le contenu légal.
- **Statique** — pas de CMS, contenu rédigé en dur ; toutes les valeurs de copie exactes (nom, email, Discord, adresses, date) vivent dans `src/lib/legal/legal-config.ts` et nulle part ailleurs.
- **Pas de bannière cookies** (cookies d'auth Supabase = strictement nécessaires, exemptés).
- **Pas de nouvelle dépendance npm.**
- **Identité éditeur (régime transition)** : éditeur & directeur de publication = **Elnop** ; contact = **contact@wizcard.xyz** ; Discord = **https://discord.gg/VkahQ2KPfA** ; hébergement = **auto-hébergé** (mail transactionnel OVH SAS, 2 rue Kellermann, 59100 Roubaix, France).
- **Rétention logs techniques** = **12 mois**.
- **Non-affiliation** : Magic: The Gathering / noms / images = propriété de **Wizards of the Coast** ; Wizcard = projet indépendant non officiel, non affilié.
- **Réclamation RGPD** = **CNIL** (www.cnil.fr).
- Conventions : composants serveur par défaut, CSS Modules, `generateMetadata` (cf. `src/app/sets/[code]/page.tsx`), composants dans `src/components/<Name>/`.
